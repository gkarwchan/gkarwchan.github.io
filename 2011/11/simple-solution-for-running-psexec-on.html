Simple solution for running Psexec on the build server

Nov 21, 2011 • Ghassan Karwchan

P.S: This post is old, and the method described here is not required any more.
Please, don't take this approach to fix the problem of running psexec remotly.

I was trying to run a deployment batch file from Hudson build server, which is using the tool psexec.exe.
The deploy process was hanging, when it is calling psexec command, and after a little bit research I found that psexec has known problems with the build servers:


  • On team city
  • On Hudson
  • and on CrsuiseControl.NET
There are alternative solutions to use alternative tools, but if you need just a simple solution that will run psexec without hanging , then keep reading.
The problem with running psexec, is the process that psexec uses to redirect the output from the remote machine to the local machine where you run the command. That redirection works when you run it from command line, but doesn't work from the build server.
So, the simple solution I used is to run an exe program that runs the batch file.
The following c# program will run the batch file that has psexec without hanging.


class Program
{
  static void Main(string[] args)
  {
     try
     {
        ProcessStartInfo psi = new ProcessStartInfo("cmd.exe", "/c " + args[0]);
        psi.UseShellExecute = true;
        psi.RedirectStandardOutput = false;
        psi.RedirectStandardError = false;
        psi.RedirectStandardInput = false;
        var p = Process.Start(psi);
        p.WaitForExit();
     }
     catch (Exception)
     {
        {
            //....
        }
        throw;
     }
  }
}

After you compile the application , and let us assume you call it "runbatch.exe"

Then from the hudson, you just call the command line:

runbatch.exe mybatchfile.bat

The disadvantages of this simple solution, is: it is suppressing the output.

So, in case you can live without the output, because in my case , it is kind of simple batch file, then you can use this method.
